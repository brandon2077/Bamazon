Bamazon Workflow

On the left, you can see that our folder structure consists of the 'bamazonSchema.sql' file containing the instructions to set up the database and tables, and the 'bamazonCustomer.js' file that will perform actions on the database. On the right, mysql console is awaiting instructions to send to the server.

On the left is the open 'bamazonSchema.sql' file, which contains the instructions that we will insert into the mysql console on the right. One option is to type or copy/paste each line from the schema into the console. Another option is to source the 'bamazonSchema.sql' file into the database by typing "source /location/of/schema.sql" into the console.

Following the schema on the left, I create the database named 'bamazon_db' using the console on the right.

Following the schema, I then tell mysql to use 'bamazon_db' for the following commands...

...create the table named 'products'

...and fill the table with product data.

On the bottom right of the mysql console, we can see the products table has been successfully created.

On the left, we are using nodejs to run 'bamazonCustomer.js', which will open the Bamazon storefront.

Upon running 'bamazonCustomer.js', a greeting message is displayed above a list of the product inventory and details. Below the list of products, a prompt awaits the users input.

The user can then input the Product ID of the desired item...

...and the amount of which they would like to buy.

10 Garden Gnomes should be enough...

Upon entering the quantity, the console displays the name, quantity, and total cost of the completed purchase. The user will then be returned to the storefront, with an updated product inventory

If the user attempts to purchase more products than we have in stock...

...they will be informed and prompted for another product.

As long as that item is in stock, the order will be fullfilled.

That's all!..for now
